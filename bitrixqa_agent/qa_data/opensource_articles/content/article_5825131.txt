Категория: Битрикс24

ТЕМА: Технические требования для установки и работы Битрикс24 (коробочная версия)

ПРОБЛЕМА:
Для корректной работы коробочной версии Битрикс24 необходимо обеспечить совместимое серверное окружение, соответствующее определенным техническим требованиям. Несоблюдение этих требований может привести к некорректной работе системы, ошибкам и проблемам с производительностью.

РЕШЕНИЕ:
1.  Веб-сервер: Использовать Apache (рекомендуется) или nginx (с самостоятельной настройкой).
2.  PHP: Минимальная версия — 8.1 и выше. Убедиться в наличии необходимых расширений (GD, PHP XML, FreeType, поддержка регулярных выражений, Zlib compression). Рекомендуется использовать акселератор PHP (OPcache).
3.  База данных: Поддерживаются MySQL (желательно 8.x) и PostgreSQL (начиная с 11.x, для Enterprise версий).
4.  Конфигурация PHP: Установить `memory_limit` (рекомендуется 256 МБ), `file_uploads = On`, настроить `upload_tmp_dir` и `upload_max_filesize`. Корректно настроить работу с сессиями (отключить `session.use_trans_sid`).
5.  Дисковое пространство: Минимум 10 ГБ.
6.  Оперативная память: Минимум 2 ГБ, рекомендуется больше.
7.  Совместимые браузеры: Использовать последние версии Chrome, Edge, Firefox, Opera, Safari.

ВАЖНО:
1.   Для работы с UTF-8 в MySQL требуется соответствующая настройка в PHP (`mbstring.func_overload=0`, `default_charset=UTF-8`).
2.   XCache требует отключения параметра `xcache.cacher = Off`.
3.   eAccelerator несовместим с PHP v5.3+.
4.   Oracle и MSSQL базы данных не поддерживаются.
5.   Рекомендуется выделять отдельные папки для хранения сессий каждого пользователя хостинга из соображений безопасности.
6.   Приведенные конфигурации серверов являются примерными и могут быть изменены в зависимости от реальных потребностей проекта.

ТЕХНИЧЕСКИЕ ДЕТАЛИ:
1.   Веб-серверы: Apache 2.4.x, Nginx 1.16.x+.
2.   PHP расширения: GD, XML, FreeType, Zlib, mbstring.
3.   PHP акселераторы: OPcache, XCache (особая настройка), APC.
4.   Базы данных: MySQL 8.x (Percona Server), PostgreSQL 11.x+.
5.   Параметры `php.ini`: `memory_limit`, `file_uploads`, `upload_tmp_dir`, `upload_max_filesize`, `session.save_path`, `session.use_trans_sid`.
6.   Способы установки `memory_limit`: `php.ini`, `dbconn.php`, `.htaccess` (при определенных условиях), `httpd.conf`.
7.   Поддерживаемые браузеры: Chrome >= 92, Edge >= 92, Firefox >= 90, Opera >= 77, Safari >= 12, Safari iOS >= 12, Chrome Android >= 93, Firefox Android >= 92.
